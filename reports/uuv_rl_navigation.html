Abstract

This report addresses the engineering challenge of designing a model-free reinforcement learning controller for an underwater unmanned vehicle (UUV). The vehicle has three degrees of freedom — yaw via differential motor thrust, forward translation via equal motor thrust, and depth via a three-state ballast pump — and is equipped with a 9-axis IMU (accelerometer, gyroscope, magnetometer) plus a pressure sensor. The RL agent must stabilize the trajectory between navigation beacon waypoints defined by magnetic bearing and depth targets, without prior knowledge of the plant transfer function, and subject to bounded external disturbances.

Across ten questions we develop: (Q1) a SysML system model; (Q2) a functional RL-environment architecture; (Q3) state and action variable spaces; (Q4) sensor fusion and observation preprocessing; (Q5) a multi-objective reward function; (Q6) a complete RL algorithm with pseudocode and Keras implementation; (Q7) convergence acceleration strategies; (Q8) training and evaluation methodology; (Q9) embedded platform selection; and (Q10) ROS integration plan. Each question follows a four-step methodology — state-of-the-art survey, comparative selection, solution planning, and solution discussion — with diagrams and mathematical derivations throughout.

Problem Statement

System Constraints

Actuators: Two DC motors driven by H-bridges with PWM for differential thrust (yaw + forward), and a ballast pump with three states (water-in, water-out, stopped) for depth control.

Sensors: 3-axis accelerometer + gyroscope (IMU), 3-axis magnetometer, and a pressure sensor for depth. Maximum sample rate: 100 Hz.

Navigation: A sequence of beacon waypoints, each specifying a target magnetic bearing and target depth. Travel time between beacons must be minimized.

Assumptions: External disturbances (currents, waves) are bounded within actuator authority. The transfer function of the plant is unknown — the RL agent must learn to control purely from sensor feedback. The UUV weighs approximately 50 kg.

Fused navigation state: Compact after sensor fusion — bearing, bearing error, heading rate, depth, depth error, depth rate, forward speed, pitch, roll. More sample-efficient since the agent focuses on control, not perception.

④ Discussion

The observation vector is designed to provide the RL agent with a PID-like information set: the error terms give proportional signals, the derivatives give rate-of-change (derivative signals), and the integral terms provide accumulated error for steady-state correction. This allows the RL agent to discover nonlinear combinations of these signals that outperform classical PID tuning, while not burdening it with raw sensor fusion.

Note on redundancy: When waypoint targets are constant between transitions, ė_ψ = d/dt(ψ_target − ψ) = −ψ̇ and similarly ė_z = −ż, making dimensions 11–12 linearly dependent on dimensions 3–4. The two pairs diverge only at waypoint transitions (where the target changes discontinuously). If observation dimensionality becomes a concern for embedded inference, these redundant dimensions can be removed with minimal impact on policy quality.

The action space mixes continuous (motor PWM) and discrete (pump). For the TD3 algorithm (Q6), we treat the pump as a continuous variable in [-1, 1] and apply rounding to {-1, 0, 1} at execution time with hysteresis thresholds at ±0.33. This avoids the complexity of hybrid action space algorithms while preserving differentiability for policy gradient updates.

All observation dimensions are normalized to [-1, 1] before feeding the neural network, using the known physical bounds listed in the parameter table. This normalization is critical for stable gradient-based training — without it, dimensions with large ranges (e.g., d_wp ∈ [0, 1000]) would dominate the loss gradients over small-range dimensions (e.g., ψ̇ ∈ [-2, 2]).

② Comparative Analysis & Selection

Complementary Filter: Combines gyro integration (high-frequency) with accelerometer/magnetometer correction (low-frequency) using a single parameter α. Sufficient for attitude estimation on a resource-constrained embedded system. The RL agent can compensate for residual sensor noise through its learned policy.

CHOSEN

Complementary Filter for attitude + direct pressure for depth — Best complexity-accuracy trade-off for embedded deployment (Q9). Attitude (pitch, roll, heading) fused from gyro + accel + mag with α = 0.98; depth from pressure with low-pass filter; heading rate from gyro z-axis. Feeds the 12D observation defined in Q3[4].

② Comparative Analysis & Selection

Sparse binary: +1 at waypoint arrival, 0 otherwise. With trajectories spanning hundreds of steps, the agent receives almost no learning signal — extremely slow convergence.

Dense shaped reward: Proportional to state improvement at every step. Critical for continuous control where immediate feedback on bearing/depth errors is essential.

Multi-objective weighted sum: Natural fit for our problem with explicit trade-offs between bearing tracking, depth tracking, energy, and time.

HER: Powerful for sparse goal-reaching but unnecessary when dense shaping is straightforward.

Potential-based shaping: Approximates optimal-policy preservation — a theoretical safeguard against reward hacking.

CHOSEN

Multi-objective weighted sum with potential-based shaping — The error terms use shaping: R_shaped = γ·Φ(s') - Φ(s), where Φ(s) = -w_ψ·|e_ψ| - w_z·|e_z| - w_d·d_wp. Combined with energy penalty and terminal bonuses, this provides dense feedback while approximating optimality preservation of the shaping component[9][5].

③ Solution Planning

where Δx = x(t) − x(t+1) (positive when error decreases)

④ Reward Strategy Discussion

Figure 6 — Reward function architecture. Weighted sum of error improvements + terminal bonus + time penalty = total scalar reward.

The weight hierarchy (w₁=3.0 > w₂=2.0 > w₃=1.0) encodes the priority: bearing accuracy is more critical than depth accuracy (the vehicle can adjust depth more easily than heading in currents), and both are more important than forward speed. The energy penalty (w₄=0.05) is intentionally small — it should discourage actuator saturation but not prevent aggressive maneuvers during course corrections. The time penalty (w₅=0.01) creates a constant negative baseline that encourages the agent to reach waypoints quickly rather than hover in a low-error state.

The potential-based shaping framework[9] approximates optimal-policy preservation when adding the shaped error-reduction terms (w₁·Δ|e_ψ| + w₂·Δ|e_z| + w₃·Δd_wp) to the base reward (energy penalty + time penalty + terminal bonuses). Formally, the formal shaping condition is F(s,s') = γ·Φ(s') − Φ(s), while our error-reduction terms use Δ|e| = |e(t)| − |e(t+1)| = Φ(s) − Φ(s'). Note: the error-reduction formulation approximates but does not strictly satisfy the formal potential-based shaping condition F(s,s') = γΦ(s') − Φ(s), as it omits the discount factor on the next-state potential. For γ ≈ 1, the approximation is tight. The energy, time, and terminal terms define the actual task objective and are not part of the shaping — they shape what is optimal, while the potential-based terms only shape how fast the agent learns it.

② Comparative Analysis & Selection

DQN: Limited to discrete actions. Discretizing PWM into a fine grid creates a combinatorial explosion (e.g., 100 × 100 × 3 = 30,000 discrete actions) — impractical[1].

DDPG: First successful continuous-control deep RL. Known to be brittle: overestimation bias in the Q-function often causes divergence[2].

TD3: Addresses DDPG's issues with twin critics, delayed policy updates, and target policy smoothing. Significantly more stable.

PPO: Very stable on-policy training but requires ~10× more samples than off-policy methods. Since UUV simulation is expensive, sample efficiency matters.

SAC: Maximum-entropy framework with automatic temperature tuning. Best sample efficiency but adds entropy tuning complexity.

CHOSEN

TD3 (Twin Delayed DDPG) — Three stability improvements: (1) twin Q-networks reducing overestimation, (2) delayed policy updates (every 2 critic updates), (3) target policy smoothing. Provides the robustness needed for safety-sensitive UUV control. Pump command treated as continuous [-1,1] with rounding at execution[7][2].

③ Solution Planning — TD3 Pseudocode[7]

PSEUDOCODEtd3_algorithm.txt
# ── TD3: Twin Delayed Deep Deterministic Policy Gradient ──

1.  Initialize critic networks Q₁(s,a; θ₁), Q₂(s,a; θ₂) with random weights
2.  Initialize actor network π(s; φ) with random weights
3.  Initialize target networks: θ₁' ← θ₁,  θ₂' ← θ₂,  φ' ← φ
4.  Initialize replay buffer B with capacity N = 100,000

5.  FOR episode = 1 TO M:
6.      Reset environment → initial state s₀
7.      FOR t = 1 TO T (max_steps_per_episode):

            # ── Exploration ──
8.          a = π(s; φ) + ε,    ε ~ 𝒩(0, σ)         # Add Gaussian noise
9.          a = clip(a, -1, +1)                       # Enforce action bounds

            # ── Environment interaction ──
10.         Execute a → observe r, s', done
11.         Store (s, a, r, s', done) in B

            # ── Training (if buffer has enough samples) ──
12.         Sample mini-batch {(sᵢ, aᵢ, rᵢ, s'ᵢ, dᵢ)} from B

            # ── Target computation ──
13.         ã' = π(s'; φ') + clip(ε̃, -c, c),   ε̃ ~ 𝒩(0, σ̃)
14.         y = r + γ · (1 - d) · min(Q₁'(s', ã'; θ₁'), Q₂'(s', ã'; θ₂'))

            # ── Critic update ──
15.         Update θ₁ by minimizing:  L₁ = 𝔼[(y − Q₁(s, a; θ₁))²]
16.         Update θ₂ by minimizing:  L₂ = 𝔼[(y − Q₂(s, a; θ₂))²]

            # ── Delayed actor update (every d steps) ──
17.         IF t mod d == 0:
18.             Update φ by maximizing: J = 𝔼[Q₁(s, π(s; φ); θ₁)]
19.             θ₁' ← τ·θ₁ + (1−τ)·θ₁'     # Soft update targets
20.             θ₂' ← τ·θ₂ + (1−τ)·θ₂'
21.             φ'  ← τ·φ  + (1−τ)·φ'

22.         s ← s'
23.         IF done: BREAK
24.     END FOR
25. END FOR
26. RETURN trained actor π(s; φ)

📊 Implementation Notes
    

  • The actor network uses 2 hidden layers of 256 ReLU units with Layer Normalization after each hidden layer, and a tanh output layer producing actions in [-1, 1] — a standard architecture for continuous control[7][2].   • Twin critics Q₁ and Q₂ receive the concatenation of state and action as input, each producing a scalar Q-value. The minimum of both is used for target computation, reducing overestimation bias.
    • 

  • Delayed policy update (d=2) means the actor is updated only every 2nd critic update, preventing the actor from exploiting transient critic inaccuracies.
    • 

  • Soft target update with τ=0.005 ensures slow tracking of the main networks, stabilizing the Bellman targets.
    • 

  • The replay buffer stores 100,000 transitions — sufficient for ~1000 episodes of 100 steps each, covering diverse state-action pairs.

② Comparative Analysis & Selection


LR scheduling: Cosine annealing reduces oscillation near optima. Zero additional memory cost. Essential baseline technique for any deep RL training.


PER: Samples transitions with high TD-error more frequently, focusing learning on surprising or difficult experiences. Particularly effective for UUV navigation where most transitions are steady-state cruising but rare events (waypoint approach, disturbance) carry the most learning signal.


Layer normalization: Stabilizes gradient magnitudes across diverse observation dimensions (angles in radians vs. depth in meters vs. distance in hundreds of meters). Low-cost improvement.


Curriculum learning: Start with easy scenarios (calm water, nearby waypoints) and progressively increase difficulty. Highly effective for navigation tasks where the agent needs basic control before handling disturbances.


Transfer learning: Pre-train on simplified dynamics, then fine-tune on high-fidelity physics. High setup cost but significant payoff.


Parallel rollouts: Linear speedup in wall-clock time. Easily implemented with vectorized environments in frameworks like Stable Baselines3.


CHOSEN
Combined: PER + Curriculum Learning + LR Scheduling — These three are complementary: PER improves which experiences to learn from, curriculum improves which scenarios to practice, and LR scheduling improves how to update weights. Expected combined speedup: 5–15× over vanilla TD3[8][5].

④ Discussion


Prioritized Experience Replay replaces uniform sampling in the replay buffer with probability-weighted sampling based on TD-error magnitude. Transitions where the critic's prediction was most wrong (high |δ|) are sampled more frequently, focusing gradient updates on the most informative experiences. The importance sampling (IS) weights compensate for the bias introduced by non-uniform sampling, with the correction factor β annealing from 0.4 to 1.0 over training.


Curriculum learning structures training into four progressive stages: (1) Calm baseline — no current, nearby waypoint at 20 m, heading error < 30°; (2) Mild disturbance — 0.2 m/s current, 50 m waypoint; (3) Moderate challenge — 0.5 m/s current, 100 m waypoint, depth changes; (4) Full difficulty — random disturbances up to 1.0 m/s, multi-waypoint sequences. The agent must achieve a threshold performance (e.g., average reward > 80% of maximum) at each stage before advancing. This prevents catastrophic forgetting by ensuring fundamental control skills are solidified before adding complexity.


Cosine learning rate scheduling with η_max = 3×10⁻⁴ and η_min = 1×10⁻⁵ over the full training horizon provides large initial steps for exploration and small final steps for fine-tuning. The cosine shape avoids the sharp transitions of step-decay schedules.


Convergence estimation: We fit an exponential model R(t) = R_∞ − (R_∞ − R₀)·exp(−t/τ) to the smoothed reward trajectory. The time constant τ characterizes convergence speed, and we declare convergence when the running average exceeds 95% of R_∞ for 100 consecutive episodes. Based on similar UUV control problems in the literature, we estimate τ ≈ 250–400 episodes with the combined approach versus τ ≈ 800–1200 episodes for vanilla TD3 — a 3–5× improvement.

② Comparative Analysis & Selection


STM32H7 MCU: Ultra-low power (< 1.5 W) with an FPU capable of running quantized neural networks (INT8 via STM32Cube.AI or TensorFlow Lite Micro). A 2-layer, 256-unit actor network quantized to INT8 requires approximately 150 KB of flash and can execute in < 1 ms on the Cortex-M7 at 480 MHz, easily meeting the 100 Hz control loop. The key advantage is deterministic real-time execution — no OS jitter from Linux. With micro-ROS, it integrates into the ROS 2 ecosystem (Q10). The STM32H7 also includes hardware DSP instructions (CMSIS-DSP) for efficient sensor fusion (Q4).


STM32N6 with NPU: The newer STM32 line with an integrated Ethos-U65 NPU accelerates INT8/INT16 inference by 10–50× over software execution. If the actor network needs to grow beyond 2 layers (e.g., for more complex environments), this provides headroom without increasing power.


FPGA + NPU co-processor: Maximum flexibility for custom quantized layer implementations. Useful when: (a) the RL agent requires a non-standard neural layer not supported by off-the-shelf hardware; (b) the quantized network needs custom bit-widths (e.g., 4-bit weights); (c) additional acceleration is needed for sensor fusion DSP alongside NN inference. However, development time is significantly higher (HDL design, timing closure).


DSP processor: Excellent for fixed-point arithmetic and sensor processing (FFT, filtering), but lacks dedicated NN acceleration. Could serve as a co-processor for sensor fusion while the MCU runs the NN, but adds board complexity.


Jetson Nano: Powerful GPU inference but draws 5–10 W versus ~1 W for the STM32H7. While motors and pump dominate total system power, the ~8 W difference is still meaningful for a 100 Wh battery over long missions. Overkill for a small 2-layer actor network. The Linux OS also introduces non-deterministic latency spikes.


Raspberry Pi 5: No hardware acceleration for NN inference. CPU-only TFLite is slower and more power-hungry than the STM32 solution for equivalent network sizes.



CHOSEN
STM32H7 MCU with INT8 quantized neural network — The 2-layer, 256-unit TD3 actor network quantized to INT8 fits in < 200 KB flash and executes in < 1 ms on the Cortex-M7 at 480 MHz, meeting the 100 Hz control loop with 90% margin. Power draw of ~1 W maximizes battery endurance. STM32Cube.AI handles the Keras-to-C conversion automatically. Hardware DSP instructions handle complementary filter computations (Q4). micro-ROS provides ROS 2 integration (Q10). The STM32N6 with Ethos-U65 NPU is a strong alternative with faster INT8/INT16 inference, but the STM32H7's more mature toolchain and broader documentation make it the lower-risk choice for initial deployment.



③ Solution Planning


The key selection criteria for this 50 kg UUV are power efficiency (100 Wh battery must also drive motors and pump), real-time determinism (100 Hz control loop with hard deadline), inference capability (2-layer 256-unit INT8 actor network), and ROS 2 compatibility (Q10). Cost and toolchain maturity are secondary but relevant for a first deployment.


The STM32H7 and STM32N6+NPU both excel on the primary criteria: sub-2W power, bare-metal real-time guarantees, and micro-ROS support. The STM32N6 offers a dedicated Ethos-U65 NPU for faster INT8/INT16 inference, giving it an edge if the network grows. However, the STM32H7 has a more mature toolchain (STM32Cube.AI), broader community documentation, and a proven track record in embedded ML inference — making it the lower-risk choice for initial deployment. The STM32N6 is the natural upgrade path.


The FPGA+NPU option provides maximum flexibility for custom quantization but demands HDL expertise and significantly longer development cycles. The Jetson Nano and Raspberry Pi 5 offer rich ecosystems and full ROS 2 support, but their Linux-based OSes cannot guarantee deterministic control loop timing, and their 3–10W compute draw — while small relative to total system power dominated by motors and pump — is unnecessary for a network that fits in < 200 KB. The DSP option lacks dedicated NN support and is better suited as a co-processor for advanced sensor fusion than as a primary inference platform.



④ Discussion — Architecture Options


Baseline configuration (recommended): A single STM32H7 runs both sensor fusion and RL inference. The Cortex-M7 at 480 MHz with FPU executes the complementary filter at < 50 μs per iteration and the INT8 actor network at < 1 ms, leaving ample margin within the 10 ms control period. STM32Cube.AI converts the Keras model (Q6) to optimized C code with INT8 quantization, typically achieving < 5% accuracy loss versus FP32.


Enhanced configuration (for growth): STM32H7 + FPGA co-processor. The FPGA handles NN inference for larger or custom-quantized networks (e.g., 4-bit weights for minimal flash usage), while the STM32 handles sensor I/O, communication, and safety monitoring. Communication via SPI at 40 MHz provides < 100 μs transfer latency. This option is warranted when: the network architecture evolves beyond what STM32Cube.AI supports, inference latency must be < 100 μs for higher control rates, or custom quantization schemes (mixed precision, ternary weights) are needed.


DSP co-processor option: If the sensor fusion pipeline grows to include more sophisticated algorithms (e.g., EKF with 15+ states, or FFT-based vibration analysis for motor diagnostics), a dedicated DSP (e.g., TI TMS320C5535 at < 0.5 W) can offload these computations from the STM32, freeing CPU cycles for NN inference. This is a modular upgrade that doesn't change the base architecture.

This report presents a complete engineering design for a reinforcement learning navigation controller for a 50 kg underwater unmanned vehicle. The TD3 algorithm (Q6) is well-suited to the continuous action space required for PWM motor control, and its twin-critic architecture provides the training stability needed for a safety-sensitive underwater platform. The key architectural insight is the separation of sensor fusion from the RL policy (Q4): by providing a pre-processed 12D observation vector containing PID-like error signals, the agent can focus on learning nonlinear control strategies rather than wasting capacity on signal processing.



The platform selection of STM32H7 with INT8 quantization (Q9) is perhaps the most impactful design decision for real-world viability. At ~1W power draw versus 5-10W for GPU-based alternatives, the MCU approach extends battery endurance from ~10 hours to over 60 hours — transforming the UUV from a short-mission demonstrator into a practical long-endurance autonomous platform. The micro-ROS integration (Q10) ensures that this power efficiency comes without sacrificing the ROS 2 ecosystem benefits of standardized communication, logging, and tooling. Combined with the three-phase training pipeline (Q8) that progressively transfers from fast Python simulation to high-fidelity Gazebo to hardware, the complete system provides a practical, cost-effective path from algorithm design to operational underwater navigation.
